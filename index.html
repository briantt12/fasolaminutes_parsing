Next Generation NLP Parser for Sacred Harp Minutes

About the Project

The Sacred Harp Minutes project is dedicated to parsing the records of Sacred Harp singings, a tradition of sacred choral music that originated in New England and was later perpetuated in the American South. These records, known as "Sacred Harp Minutes," document the leaders and songs in each singing session. The minutes are stored in a database that researchers and singers can query for historical insights into the Sacred Harp tradition. The goal of this project is to develop a next-generation natural language processing (NLP) parser capable of accurately parsing these minutes, while handling various issues such as OCR errors, misspelled names, and ambiguous entries. The parser will integrate with the existing SQL-based database and provide improved usability for scholars and researchers.

Project Goal

  • Accurately parse leader names, song numbers, and titles
  • Handle OCR errors and ambiguous/misspelled names
  • Integrate with an existing SQL-based database

Project Requirements

Dependencies

  • Regex: Extract song page numbers and names
  • Ollama: LLM for validating names in context
  • RapidFuzz: Fuzzy matching for misspelled names
  • SQLite: Database integration
  • Pandas: Data handling and analysis

Software & Tools

  • Python 3.x
  • SQLite
  • LLM API access via Ollama

Project Evolution

Original Approach: parse_minutes.py

Used regular expressions for basic extraction. Limited handling of errors and name diversity.

Second Approach: aman_parse_minutes.py

Introduced spaCy for NER and RapidFuzz for fuzzy matching. Improved performance but had context limitations.

Current Approach: llm_parse_minutes.py

  • Regex for patterns
  • RapidFuzz for fuzzy matching
  • Ollama LLM for context-based validation

This approach significantly improves name recognition and parsing accuracy.

Methodology

  1. Regex: Used for basic parsing of songs and names.
  2. Fuzzy Matching: Identifies misspelled names using RapidFuzz.
  3. Contextual Validation: LLM (Ollama) verifies real names vs roles or orgs.
  4. Benchmarking: Evaluates accuracy, precision, and recall.

Results and Value

  • Improved name validation and accuracy using LLM
  • Seamless integration with SQL database
  • Benchmarking to quantify parsing improvements
